при заходе на страницу index.php обращается к классу controller, в нём вызывается класс user controller, и если существует - создаётся класс tasks controller

user определяет, залогинен ли пользователь, если нет - возвращает в user view false, после чего тот отображает на странице поля для логина/пароля и кнопку входа
если пользователь залогинен - обращается к model user, который возвращает данные о пользователе, а также передаёт в user view true, после чего тот отображает имя пользователя и кнопку выхода

tasks controller существует, только если пользователь залогинен, нужен для добавления, удаления и смены статуса задач

user model отвечает за логин(регистрацию)/логаут
tasks model отвечает за вывод задач, если tasks controller существует, передаёт данные о задачах в tasks view

user view если пользователь есть - выводит имя пользователя, полученное из контороллера и кнопку выхода, если пользователя нет - выводит поля для логина/кнопки выхода
tasks view, срабатывает если пользователь есть - выводит поле для добавления задач, и сами задачи, полученные из tasks model, и кнопки смены статуса/удаления этих задач
